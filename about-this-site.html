About This Site

Updated 2026-06-13

Tracking/Privacy

I don't log traffic on this site in any way.
I do not use cookies, tracking pixels, or anything of that ilk.
I never have and I never will.

Javascript

As of this writing, I use Javascript exclusively for rendering the sidebar, and for no other purpose. At some point in the future, I plan to use a 3rd-party open-source Javascript module for rendering sheet music. I have no other plans to use Javascript.

Reverse-Engineering

This website is designed to be, first and foremost, readable! This applies both to the interface you see and to the code that you (usually) don't. I, personally, think that one of the best ways to learn to code is to reverse-engineer other people's code. You are more than welcome to do that to this website, and in fact, I have structured the HTML in such a way as to make it more readable for you, including adding comments in the code when I do something obscure (I try not to do that often). You can also check out the source code either with your browser's inspector tool or at github.com/PurpleRabbit75/purplerabbit75.github.io

Design Decisions

Here are some things I'm thinking through as I design this website.



Thing #1: Designed To Last

As you can read in the footer, this page is designed to last! There are several principles of "design-to-last", all of which you can read about on Jeff Huang's excellent webpage (linked). The one that's having the most influence on me at the moment is his call to return to vanilla HTML/CSS. I'm going to try to keep this codebase as simple as I possibly can for the most part, with the notable exception of the "Music" section. Along with that, I'm going to try to keep it very very readable, both for my sake and for yours.

With the perpetual exception of the "Music" section, I'm also going to avoid hotlinking when I can. This means that, if my site needs a given resource to function, the code for that needs to all be right here, specified in local paths within the site directory. Hopefully, this should also make reverse-engineering this easier, since you can always git clone the repository and it should function just fine locally, without any internet (the solitary exception is, I'm afraid, the sidebar, which tends to break on local downloads due to security policies. I'm working on it...). Also, that should make this site easier to archive!

I intend to stick with web-native fonts and vector-graphic images when possible.


Thing #2: Principles of Good Code

There's no reason that just because this is a website, its source code has to be unreadable and awful. Instead, I think I prefer a world where there isn't an algorithm stripping out all the whitespace on here to save a few bits of bandwidth. Yes, and let's keep the comments too, where relevant. And I'm going to indent indent indent!


Thing #3: Local Hosting & Fonts

This website is typeset in Noto Serif, using .woff2 files downloaded from GitHub using gwfh.mranftl.com. All fonts are hosted locally--there are no Google APIs anywhere on this site.
The favicon is an edited version of a public domain file available at Wikimedia Commons I edited it using inkscape and converted it to an .ico using CloudConvert.


This Site Poisons Rude Bots

Any person wishing to scrape this site has two choices: they may choose to comply with my robots.txt file, or not. Robots that comply with my instructions there may live a long and happy life scraping the web. Robots that fail to comply will be poisoned. If you develop or deploy web scrapers, thank you for respecting robots.txt files to make the internet a kinder place!
